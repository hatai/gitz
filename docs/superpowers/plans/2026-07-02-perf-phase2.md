# パフォーマンスチューニング Phase 2 実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or executing-plans。タスク単位で実装。

**Goal:** Phase 2（所有権 / arena 統合）を実装し、`topology.parse` の O(N×P) 個別 dupe と `handleLogLoaded` の substrate clone を廃止し、10万コミットでのメモリ/alloc 負荷を線形改善する。

**Tech Stack:** Zig 0.16.0 + zigzag v0.1.5（固定）。テストは実装 `.zig` 内 `test {}`・`std.testing.allocator` 必須。

**Spec:** `docs/superpowers/specs/2026-06-30-perf-tuning-design.md`（rev.2・§6 Phase 2・§8 所有権例外）。未固定だった §6.2 backing / §6.3 SBO 上限 / §6.4 延期は本 plan で固定。

**Phase 0 データ（設計根拠）:** frontier max = 1-3（1000コミット・6 profile）・topology.parse = 4016 allocs / peak 200KB（1000commits）・substrate clone は `handleLogLoaded:606`。

## Task 1: §6.1 — `update(*Model, *Msg)` API 変更（codex BLOCKER 1 解消）

**Files:** Modify: `src/update.zig`・`src/main.zig`

**設計（spec §6.1）:**
- 純粋 reducer `pub fn update(model: *Model, msg: *Msg) !AppCmd` へ（ポインタ渡し）。
- **zigzag trait `RuntimeModel.update`（`main.zig:329`）は変更しない**（zigzag 要求 signature）。内部から純粋 reducer を `&msg` で呼ぶ。
- `main.step`（`main.zig:233` 他 4 箇所）は既に `var msg` 化済み→ `update.update(&app.model, &msg)` へ。
- **テスト呼出の一括変換（171箇所）**: `var c = try update(&m, .key_down);` →
  ```zig
  var msg = Msg{ .key_down = {} };
  defer msg.deinit(a);
  var c = try update(&m, &msg);
  ```
  ペイロード付き（`.status_loaded = &.{}` 等）も同様。コンパイラが網羅性を強制する。

## Task 2: §6.2 — TopologySubstrate arena + clone 廃止・move（codex BLOCKER 2 / MAJOR 6 解消）

**Files:** Modify: `src/git/topology.zig`・`src/messages.zig`・`src/model.zig`・`src/update.zig`

**設計（spec §6.2・設計セッション固定）:**
- **backing = `std.heap.ArenaAllocator`**（`arena: std.heap.ArenaAllocator` フィールド）。
  - `parse`: 全 hash/parents 文字列を arena.allocator() へ（O(N×P) dupe → 実質 O(1) アロケーション群・arena が一括管理）。
  - `hash_index`: **gpa で保持**（keys は arena 内 slice を借用・HashMap.deinit は構造体のみ解放）。deinit 順序: `hash_index.deinit()` → `arena.deinit()`（arena が先だと HashMap 解放が freed memory を触るため逆順厳守）。
- **`clone` 廃止**: `Msg.LogLoaded.substrate` → Model へ **take（move）**。
  - `Msg.LogLoaded.takeSubstrate() ?TopologySubstrate`: substrate フィールドを null 化して値を返す。
  - `Msg.deinit`（`messages.zig:145`）は null 済みなら解放しない（disarm）。
- **`handleLogLoaded:604-611`**: `sub.clone(...)` → `ll.takeSubstrate()` へ。take は stale reject 通過後・`replaceLogCommits` 成功後・`setTopologySubstrate` 直前に限定。take 後 projection/computeAll 失敗時は `model.clearTopologySubstrate()` で確実解放。
- **OOM safety**: `checkAllAllocationFailures` で `handleLogLoaded filter+substrate` 全 alloc 失敗点で「リーク無し・二重 free 無し・Model 半端状態無し」を固定。

- [ ] **Step 1: TopologySubstrate へ arena backing 追加・parse を arena 化・deinit 順序固定**
- [ ] **Step 2: clone 廃止・`Msg.LogLoaded.takeSubstrate()` 追加・deinit disarm**
- [ ] **Step 3: handleLogLoaded を take へ・take 後失敗時 clearTopologySubstrate**
- [ ] **Step 4: OOM safety テスト（checkAllAllocationFailures for handleLogLoaded filter+substrate）**
- [ ] **Step 5: 既存 substrate/clone テストを新所有権モデルへ更新**
